## Troubleshooting Reference

```
COMMON MISTAKES AND HOW TO AVOID THEM:

1. "Tests pass but release-tree has wrong content"
   → You forgot to rewrite paths inside the copied text files. Grep the release-tree
     for old patterns: grep -r "skeleton.md\|complete-reference.md\|L2-semantic\|openwrt-condensed-docs" release-tree/

2. "05e crashes on a missing file"
   → Some modules may not have types/ or may not have sharded parts. Always check
     existence before copying optional files. Use os.path.exists() guards.

3. "Smoke tests fail after adding 05e to the pipeline list"
   → The smoke runner executes scripts in order. 05e must come AFTER 05d and BEFORE 06.
     Check the exact position in POST_EXTRACT_PIPELINE.

4. "Tests fail with ENABLE_RELEASE_TREE=false after Phase 4 changes"
   → You broke the old code path. Every modified script must have a clean if/else
     branch. The false path must produce EXACTLY the same output as before your changes.
     Diff the output before and after.

5. "Path separator issues on Windows"
   → Always use os.path.join(). Never write "release-tree/ucode/map.md" as a path.
     In generated TEXT content (llms.txt, AGENTS.md), forward slashes are correct
     and expected — they are URLs, not filesystem paths.

6. "shutil.copy2 fails with permission error on Windows"
   → Target directory may not exist. Always os.makedirs(target_dir, exist_ok=True)
     before copying.

7. "Module discovery picks up L1-raw and L2-semantic as modules"
   → Filter your module list. A module is a directory in OUTDIR that contains
     {module}-skeleton.md (current) or map.md (V5a). L1-raw and L2-semantic are
     not modules. Neither are files (llms.txt, AGENTS.md, etc.).

8. "index.html has broken links after path rewriting"
   → index.html uses href attributes. Your replacement must handle both
     href="./openwrt-condensed-docs/..." and visible link text. Read the current
     index.html before writing replacements — understand its structure first.

FILES YOU SHOULD NEVER NEED TO TOUCH:
  - .github/scripts/openwrt-docs4ai-01-clone-repos.py
  - .github/scripts/openwrt-docs4ai-02a-scrape-wiki.py through 02h
  - .github/scripts/openwrt-docs4ai-03-normalize-semantic.py
  - .github/scripts/openwrt-docs4ai-04-generate-ai-summaries.py
  - lib/ai_store.py, lib/ai_enrichment.py
  - data/base/*, data/override/*
  - tests/pytest/pytest_00_pipeline_units_test.py
  - tests/pytest/pytest_03_wiki_corpus_sanity_test.py
  - tests/pytest/pytest_04_wiki_scraper_test.py
  - tests/pytest/pytest_05_manage_ai_store_cli_test.py

IF IN DOUBT:
  - Read the file before changing it.
  - Run tests after every change, not just at the end.
  - Check both flag states.
  - Ask the human operator rather than guessing.
```
